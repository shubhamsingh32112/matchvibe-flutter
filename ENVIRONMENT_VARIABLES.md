# Environment Variables Used in Flutter App

This document lists all environment variables and configuration values used throughout the Flutter app codebase.

## Build-Time Environment Variables (--dart-define)

These are set at compile time using `--dart-define` flag:

### 1. `USE_EMULATOR_IP`
- **Type**: `bool`
- **Default**: `false`
- **Usage**: Used to determine if running on Android emulator (uses `10.0.2.2` IP)
- **Location**: `lib/core/constants/app_constants.dart` (lines 49, 82, 110, 137)
- **How to set**: `flutter run --dart-define=USE_EMULATOR_IP=true`

### 2. `USE_SIMULATOR_IP`
- **Type**: `bool`
- **Default**: `true`
- **Usage**: Used to determine if running on iOS simulator (uses `localhost`)
- **Location**: `lib/core/constants/app_constants.dart` (lines 57, 88, 116, 143)
- **How to set**: `flutter run --dart-define=USE_SIMULATOR_IP=false`

## Firebase Configuration

Firebase is configured via generated files (not environment variables):

### Android
- **File**: `android/app/google-services.json`
- **Generated by**: `flutterfire configure`
- **Contains**: Firebase project configuration, API keys, etc.

### iOS
- **File**: `ios/Runner/GoogleService-Info.plist`
- **Generated by**: `flutterfire configure`
- **Contains**: Firebase project configuration, API keys, etc.

### Firebase Initialization
- **Location**: `lib/main.dart` (line 28)
- **Note**: Requires `firebase_options.dart` file (generated by `flutterfire configure`)

## Backend API URLs (Constructed from Constants)

These URLs are dynamically constructed based on platform and constants:

### 9. `baseUrl` (Backend API Base URL)
- **Type**: `String` (getter)
- **Usage**: Base URL for all REST API calls
- **Location**: `lib/core/constants/app_constants.dart` (lines 33-67)
- **Constructed from**: `BACKEND_PORT`, `API_VERSION`, `LAN_IP_ADDRESS`, `USE_USB_TUNNEL`, `USE_EMULATOR_IP`, `USE_SIMULATOR_IP`
- **Examples**:
  - Web: `http://localhost:3000/api/v1`
  - USB Tunnel: `http://localhost:3000/api/v1`
  - Android Emulator: `http://10.0.2.2:3000/api/v1`
  - iOS Simulator: `http://localhost:3000/api/v1`
  - Real Device: `http://192.168.1.5:3000/api/v1`

### 10. `socketUrl` (Socket.IO URL)
- **Type**: `String` (getter)
- **Usage**: WebSocket URL for Socket.IO real-time connections
- **Location**: `lib/core/constants/app_constants.dart` (lines 71-96)
- **Constructed from**: Same constants as `baseUrl` but without `/api/v1` suffix
- **Used in**: `lib/core/services/socket_service.dart` (line 90)

### 11. `healthCheckUrl` (Health Check URL)
- **Type**: `String` (getter)
- **Usage**: Health check endpoint for backend connectivity testing
- **Location**: `lib/core/constants/app_constants.dart` (lines 99-124)
- **Constructed from**: Same constants as `baseUrl` but with `/health` endpoint

### 12. `websiteBaseUrl` (Website Base URL)
- **Type**: `String` (getter)
- **Usage**: Base URL for static/legal pages (Privacy Policy, etc.)
- **Location**: `lib/core/constants/app_constants.dart` (lines 127-151)
- **Constructed from**: `WEBSITE_PORT` and network constants

## Recommendations

1. **Move hardcoded values to environment variables**:
   - Use `--dart-define` for build-time values
   - Or use a `.env` file with `flutter_dotenv` package
   - Or use platform-specific configuration files

2. **Create environment-specific configurations**:
   - Development: `localhost`, `192.168.1.5`
   - Staging: Staging server URL
   - Production: Production server URL

3. **Document required environment variables**:
   - Create `.env.example` file
   - Document in README.md

4. **Consider using flutter_dotenv**:
   ```yaml
   dependencies:
     flutter_dotenv: ^5.1.0
   ```
   Then load from `.env` file at runtime.
